\subsection{Vectorization Engine}

Each transaction is converted into a 16-dimensional feature vector through deterministic feature extraction:

\[
\vec{v}_{tx} = \begin{pmatrix}
\text{gasNorm} \\
\text{valueLog} \\
\text{priorityFeeRatio} \\
\text{methodEntropy} \\
\text{senderEntropy} \\
\text{receiverEntropy} \\
\text{contractDepth} \\
\text{inputDataSize} \\
\text{nonceNorm} \\
\text{accessListSize} \\
\text{blobCount} \\
\text{timingOffset} \\
\text{positionInBlock} \\
\text{valueVelocity} \\
\text{gasEfficiency} \\
\text{interactionComplexity}
\end{pmatrix} \in \mathbb{R}^{16}
\]

Each feature is normalized to $[-1, 1]$ using chain-specific historical statistics, enabling direct cross-chain comparison.

\subsection{Folding Algorithm}

The folding algorithm aggregates all transaction vectors within a block into a single 96-dimensional fingerprint. This captures not just averages, but the complete statistical distribution of behavioral patterns:

\[
\vec{F}_{block} = \text{fold}(\{\vec{v}_1, \vec{v}_2, \ldots, \vec{v}_n\})
\]

The 96 dimensions are organized as:

\begin{itemize}[leftmargin=*]
    \item \textbf{Dimensions 1-16:} Mean of each feature across all transactions ($\mu$)
    \item \textbf{Dimensions 17-32:} Variance of each feature ($\sigma^2$)
    \item \textbf{Dimensions 33-48:} Skewness of each feature distribution
    \item \textbf{Dimensions 49-64:} Kurtosis (tail behavior)
    \item \textbf{Dimensions 65-80:} Cross-feature covariance (top principal components)
    \item \textbf{Dimensions 81-96:} Temporal evolution within block (first half vs. second half deltas)
\end{itemize}

This fingerprint encodes the block's complete ``behavioral personality'' in a fixed-size vector suitable for machine learning and similarity search.

\subsection{Product Quantization with Learned Codebook}

The 96-dimensional fingerprint is compressed using Product Quantization (PQ) with a codebook trained via k-means on historical block data:

\[
\text{PQ}(\vec{F}) = (c_1, c_2, \ldots, c_6), \quad c_i \in \{0, 1, \ldots, 255\}
\]

The fingerprint is split into 6 subspaces of 16 dimensions each. Each subspace has 256 learned centroids optimized to minimize reconstruction error across the training corpus.

\textbf{Reconstruction residual:}
\[
r = \|\vec{F} - \hat{\vec{F}}\|_2 \quad \text{where} \quad \hat{\vec{F}} = \text{decode}(c_1, \ldots, c_6)
\]

The residual $r$ is a critical signal: high residuals indicate blocks that don't fit learned patterns, often preceding market regime changes.

\subsection{Kernel Density Estimation and Hotzones}

The system maintains a continuous density map of the 96-dimensional fingerprint space using Kernel Density Estimation (KDE):

\[
\hat{f}(\vec{x}) = \frac{1}{nh^{96}} \sum_{i=1}^{n} K\left(\frac{\vec{x} - \vec{F}_i}{h}\right)
\]

Dense regions are identified as \textbf{hotzones}, representing clusters of similar behavioral patterns. The system tracks:

\begin{itemize}[leftmargin=*]
    \item \textbf{Hotzone centroids:} The center of each behavioral cluster
    \item \textbf{Hotzone density:} How concentrated the cluster is
    \item \textbf{Hotzone membership:} Which blocks belong to which cluster
    \item \textbf{Hotzone velocity:} How centroids drift over time
\end{itemize}

When a new block's fingerprint lands far from any hotzone, it triggers an anomaly signal.

\subsection{Hypergraph Structure}

Relationships between hotzones are encoded in a weighted hypergraph:

\begin{itemize}[leftmargin=*]
    \item \textbf{Nodes:} Hotzones (behavioral clusters)
    \item \textbf{Pairwise edges:} Connect hotzones with overlapping membership
    \item \textbf{Triple edges:} Connect triplets of hotzones that frequently co-occur in sequence
    \item \textbf{Quad edges:} Higher-order relationships for complex patterns
\end{itemize}

Edge weights encode:
\[
w_{ij} = \alpha \cdot \text{overlap}(H_i, H_j) + \beta \cdot \text{transition\_freq}(H_i \to H_j) + \gamma \cdot \text{density\_product}
\]

The Graph Neural Network (GNN) learns structural patterns in this hypergraph that predict future edge formations and hotzone migrations.

\subsection{Anomaly Detection Engine}

The anomaly detection engine combines multiple ML models:

\subsubsection{Isolation Forest}
Trained on historical fingerprints to identify statistical outliers:
\[
s_{IF}(\vec{F}) = 2^{-\frac{E[h(\vec{F})]}{c(n)}}
\]
where $h(\vec{F})$ is the path length in the isolation tree and $c(n)$ is the normalization factor.

\subsubsection{Autoencoder Reconstruction}
A deep autoencoder compresses fingerprints to an 8-dimensional bottleneck and reconstructs:
\[
s_{AE}(\vec{F}) = \|\vec{F} - \text{Decoder}(\text{Encoder}(\vec{F}))\|_2
\]
High reconstruction error indicates the fingerprint doesn't match learned ``normal'' patterns.

\subsubsection{Combined Anomaly Score}
The final anomaly score fuses multiple signals:
\[
s_{anomaly} = w_1 \cdot s_{IF} + w_2 \cdot s_{AE} + w_3 \cdot r_{PQ} + w_4 \cdot (1 - \text{hotzone\_density})
\]
Weights are learned via gradient descent on historical regime change labels.

\subsection{Predictive Models}

\subsubsection{LSTM Sequence Model}
A bidirectional LSTM processes sequences of fingerprints to predict future behavior:

\begin{itemize}[leftmargin=*]
    \item \textbf{Input:} Last 20 block fingerprints (96-dim each)
    \item \textbf{Architecture:} 2-layer BiLSTM with 128 hidden units
    \item \textbf{Output heads:}
    \begin{itemize}
        \item Predicted next fingerprint (96-dim)
        \item Predicted tag probabilities (15-dim)
        \item Predicted anomaly probability (scalar)
        \item Confidence estimate (scalar)
    \end{itemize}
\end{itemize}

\subsubsection{Transformer Attention Model}
For longer-range dependencies, a transformer model attends over 100-block windows:

\begin{itemize}[leftmargin=*]
    \item \textbf{Architecture:} 4-layer transformer with 8 attention heads
    \item \textbf{Positional encoding:} Learned embeddings + sinusoidal time
    \item \textbf{Output:} Distribution over possible future fingerprint clusters
\end{itemize}

\subsubsection{Historical Pattern Matching}
The system maintains a vector database of all historical fingerprint sequences. When the current sequence shows high similarity to a past sequence, the system retrieves what happened next:

\[
\text{similar\_history} = \text{top-}k\left(\text{cosine}(\text{seq}_{current}, \text{seq}_{historical})\right)
\]

This enables ``we've seen this pattern before'' signals with concrete historical references.

\subsection{Cross-Chain Correlation Engine}

Because fingerprints are normalized to the same 96-dimensional space, cross-chain analysis is native:

\subsubsection{Lead/Lag Detection}
For each pair of chains, the system computes time-lagged correlations:
\[
\rho_{lag}(A, B, \tau) = \text{corr}(\vec{F}^A_t, \vec{F}^B_{t+\tau})
\]

When ETH fingerprints predict AVAX fingerprints with a consistent lag, this indicates arbitrage opportunities or behavioral contagion.

\subsubsection{Divergence Detection}
When chains that normally correlate start diverging:
\[
\text{divergence}_{AB} = \|\vec{F}^A_t - \vec{F}^B_t\|_2 - \mu_{historical}
\]
Significant positive divergence signals chain-specific events.

\subsection{Cryptographic Verification Layer}

All fingerprints are cryptographically committed:

\subsubsection{BLAKE3 Commitments}
\[
\text{digest} = \text{BLAKE3}(\text{chain} \| \text{height} \| \vec{F} \| \text{codes} \| \text{tags})
\]

\subsubsection{Halo2 Zero-Knowledge Proofs}
The entire folding pipeline is implemented as a Halo2 circuit:
\begin{itemize}[leftmargin=*]
    \item \textbf{Public inputs:} Block height, chain ID, commitment digest
    \item \textbf{Private witness:} Raw transactions, intermediate computations
    \item \textbf{Proven statements:}
    \begin{itemize}
        \item Fingerprint was correctly computed from transactions
        \item PQ codes match the fingerprint
        \item Commitment binds all values
    \end{itemize}
\end{itemize}

This enables trustless verification: trading partners can verify that fingerprints were honestly computed without trusting YYSFold's infrastructure.
